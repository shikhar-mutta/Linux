FILE AND DIRECTORY MANAGEMENT
================================================================================

1. ls - List directory contents
   Use Case: Display files and directories in the current directory
   Example: ls

   Variations of ls command:
   
   a) ls -a (or --all)
      Use Case: Show all files including hidden files (starting with .)
      When to use: When you need to see configuration files like .bashrc, .gitignore
   
   b) ls -l (or --long)
      Use Case: Display detailed list view with permissions, owner, size, date
      When to use: When you need file permissions, size, and modification time details
   
   c) ls -la (or -al)
      Use Case: Combine both -l and -a to show all files with detailed information
      When to use: Complete file listing including hidden files with full details
   
   d) ls -lh (or --human-readable)
      Use Case: Show file sizes in human-readable format (KB, MB, GB)
      When to use: When file sizes are too large and you want easier reading
   
   e) ls -R (or --recursive)
      Use Case: List contents of all subdirectories recursively
      When to use: When you need to see directory structure at multiple levels
   
   f) ls -r (or --reverse)
      Use Case: Reverse the sort order of files
      When to use: When you want descending alphabetical order instead of ascending
   
   g) ls -S (or --sort=size)
      Use Case: Sort files by size in descending order
      When to use: To find large files taking up disk space
   
   h) ls -t (or --sort=time)
      Use Case: Sort files by modification time (newest first)
      When to use: To see recently modified files first
   
   i) ls -rt
      Use Case: Sort files by modification time (oldest first)
      When to use: To see oldest files first (reverse time order)
   
   j) ls *.txt
      Use Case: List only files matching a pattern
      When to use: To filter and show only specific file types
   
   k) ls -1 (numeric one)
      Use Case: List files one per line
      When to use: Better for scripting or when you want simple single-column output
   
   l) ls -m
      Use Case: List files separated by commas
      When to use: When you want comma-separated list format
   
   m) ls -F (or --classify)
      Use Case: Append indicators to file types (/ for directories, * for executables, etc.)
      When to use: To quickly identify file types (files, directories, links, etc.)
   
   n) ls -i (or --inode)
      Use Case: Display inode number for each file
      When to use: When working with file systems or identifying hard links
   
   o) ls -L (or --dereference)
      Use Case: List symbolic link targets instead of the link itself
      Example: ls -L
      When to use: To see what symbolic links point to rather than the link path
      
   p) ls -lhS
      Use Case: Sort by file size with human-readable format and detailed view
      When to use: To quickly find largest files in a directory
   
   q) ls -lht
      Use Case: Sort by modification time with human-readable format and detailed view
      When to use: To see recently modified files with full details

2. pwd - Print working directory
   Use Case: Display the full path of the current directory you're working in
   Example: pwd

   Variations of pwd command:
   
   a) pwd (basic)
      Use Case: Display the current working directory path
      When to use: Most common usage - to know your current location in the file system
   
   b) pwd -L (or --logical)
      Use Case: Display logical path (follows symbolic links as they appear)
      When to use: When you want the path as it appears with symbolic links
   
   c) pwd -P (or --physical)
      Use Case: Display physical path (resolves all symbolic links to actual paths)
      When to use: When you need the actual directory location, not symlink references
   
   d) pwd > file.txt
      Use Case: Save current directory path to a file
      When to use: When you want to save your current location for later reference
        
3. cd - Change directory
   Use Case: Navigate to different directories in the file system
   Example: cd /home/user/Documents

   Variations of cd command:
   
   a) cd (no arguments)
      Use Case: Change to home directory
      When to use: Quick way to return to home directory from anywhere
   
   b) cd ~ (tilde)
      Use Case: Explicitly navigate to home directory
      When to use: Same as 'cd', useful in scripts for clarity
   
   c) cd /path/to/directory (absolute path)
      Use Case: Navigate using full path from root
      When to use: When you know the exact full path of target directory
   
   d) cd ./subfolder (relative path)
      Use Case: Navigate to subfolder of current directory
      When to use: When moving to folders inside current directory
   
   e) cd .. (parent directory)
      Use Case: Navigate to parent directory (one level up)
      When to use: Go up one level in directory hierarchy
   
   f) cd ../.. (multiple levels up)
      Use Case: Navigate multiple levels up the directory tree
      When to use: Go up 2 or more directory levels
   
   g) cd - (previous directory)
      Use Case: Switch to the previous directory
      When to use: Toggle between two directories
      Note: Requires OLDPWD to be set (after first directory change)
   
   h) cd ~username (another user's home)
      Use Case: Navigate to another user's home directory
      When to use: Access another user's home directory (if permissions allow)
   
   i) cd $HOME
      Use Case: Navigate using environment variable
      When to use: In scripts where you want explicit home directory reference
   
   j) cd /
      Use Case: Navigate to root directory
      When to use: Go to system root directory
   
   k) cd /tmp
      Use Case: Navigate to temporary directory
      When to use: Work with temporary files and scripts
   
   l) cd ~ && ls (combine with other commands)
      Use Case: Change directory and immediately execute command
      When to use: Navigate and perform action in one command
   
   q) cd "directory with spaces"
      Use Case: Navigate to directory with spaces in name
      Example: cd "My Documents" or cd My\ Documents
      When to use: Handle directories with spaces using quotes or escaping
   
   r) cd /home/*/Desktop (wildcard expansion)
      Use Case: Navigate using wildcard patterns
      When to use: Match directories using pattern matching
   
   t) cd $(pwd)
      Use Case: Stay in current directory (confirms current location)
      When to use: Debugging or in scripts to validate location

4. mkdir - Create directory
   Use Case: Create new folders/directories in the file system
   Example: mkdir folder_name
   
   Variations with Use Cases:
   
   a) mkdir folder_name
      Use: Create a single directory in current location
      When to use: Basic folder creation
   
   b) mkdir /home/user/Projects/NewProject
      Use: Create directory at specified absolute path
      When to use: Create folder in specific location from anywhere
   
   c) mkdir mkdir ./Documents/MyFolder
      Use: Create directory relative to current location
      When to use: Create nested folders starting from current directory
   
   d) mkdir -p /path/with/nested/folders
      Use: Create directory and all parent directories if they don't exist
      When to use: Create entire directory structure in one command
   
   e) mkdir folder1 folder2 folder3
      Use: Create multiple directories in one command
      When to use: Create several folders at once
   
   f) mkdir -m 755 folder_name
      Use: Create directory with specific permissions
      Permissions: 755 = rwxr-xr-x (owner read/write/execute, others read/execute)
   
   g) mkdir -m 700 private_folder
      Use: Create directory accessible only by owner
      When to use: Create private directories for sensitive data
   
   h) mkdir -v folder_name
      Use: Show messages as directories are created
      When to use: See confirmation of directory creation
   
   i) mkdir -p -v /nested/directory/structure
      Use: Create nested directories with verbose output
      When to use: Create complex structure with confirmation
   
   j) mkdir {folder1,folder2,folder3}
      Use: Create multiple directories using shell expansion
      When to use: Create multiple similar named directories
   
   k) mkdir -p project/{src,build,docs,tests}
      Use: Create directory structure with subdirectories
      When to use: Initialize project structure scaffolding
   
   l) mkdir $(date +%Y-%m-%d)
      Use: Create directory with current date as name
      When to use: Create dated folders for daily logs/backups
   
   m) mkdir -p ~/NewProject && cd ~/NewProject
      Use: Create directory and immediately enter it
      When to use: Set up and work in new directory in one go
   
   n) mkdir test_$(date +%s)
      Use: Create uniquely named directories with timestamp
      When to use: Create multiple test directories with unique names
   
   o) for i in {1..5}; do mkdir folder_$i; done
      Use: Create multiple numbered directories using loop
      When to use: Create series of numbered directories
   
   p) if ! [ -d folder ]; then mkdir folder; fi
      Use: Create directory only if it doesn't exist
      When to use: Safe creation in scripts to avoid errors

5. rmdir - Remove empty directory
   Use Case: Delete empty directories
   Example: rmdir my_folder

   Variations with Use Cases:
   
   a) rmdir folder_name
      Use: Remove a single empty directory in current location
      When to use: Basic directory removal
      Note: Only works if directory is completely empty
   
   b) rmdir /path/to/folder
      Use: Remove directory at specified absolute path
      When to use: Remove directory from anywhere in the file system
   
   c) rmdir folder1 folder2 folder3
      Use: Remove multiple empty directories at once
      When to use: Delete several empty folders in one command
   
   d) rmdir -p /path/to/nested/folder
      Use: Remove directory and empty parent directories
      When to use: Clean up nested directory structure when all are empty
      Note: Stops if it encounters non-empty directory
   
   e) rmdir -p --ignore-fail-on-non-empty folder
      Use: Ignore errors if directories are not empty
      When to use: Script execution without stopping on errors
   
   f) rmdir -v folder_name
      Use: Show messages when directory is removed
      When to use: Confirmation of directory deletion

   g) rmdir folder/subfolder/nested
      Use: Remove nested directory with relative path
      When to use: Remove subdirectory relative to current location
   
   h) for folder in folder1 folder2 folder3; do rmdir "$folder"; done
      Use: Remove multiple folders using loop
      When to use: Iterate through list of directories to delete   
   
   i) find . -type d -empty -exec rmdir {} \;
      Use: Find and remove all empty directories recursively
      When to use: Clean up all empty folders in directory structure
   
   j) rmdir ~/.cache/empty_dir
      Use: Remove directory from home directory using tilde
      When to use: Clean up home directory subdirectories
   
   k) rmdir ./folder
      Use: Remove directory using relative path with dot notation
      When to use: Remove relative to current location 

6. rm - Remove files
   Use Case: Delete files and directories
   Example: rm -r my_folder (recursive deletion)

   Variations of rm command:

   a) rm <filename>
      Use Case: Remove a single file
      When to use: When you want to delete a specific file
   
   b) rm -f read_only_file.txt (or --force)
      Use Case: Force remove a file without prompting
      When to use: When you want to remove files without confirmation or when files have restricted permissions
   
   c) rm -i file.txt (or --interactive)
      Use Case: Prompt before deleting each file
      When to use: When you want confirmation before removing files to avoid accidental deletion
   
   d) rm -r my_folder (or --recursive)
      Use Case: Remove a directory and all its contents recursively
      When to use: When you want to delete a directory with files and subdirectories
   
   e) rm -rf old_project (or --recursive --force)
      Use Case: Force remove a directory and all contents without prompting
      When to use: When you want to quickly delete large directories without confirmation (use with caution!)
   
   f) rm -ri <directory> (or --recursive --interactive)
      Use Case: Recursively remove directory with prompts for each item
      When to use: When deleting large directories but you want to be careful about what's removed
   
   g) rm *.txt (deletes all text files)
      Use Case: Remove all files matching a pattern
      When to use: When you want to delete all files of a specific type
   
   h) rm -v <filename> (or --verbose)
      Use Case: Display files being deleted
      When to use: When you want to see what files are being removed
   
   i) rm -rv <directory> (or --recursive --verbose)
      Use Case: Recursively remove directory while showing all deleted items
      When to use: When you want visual feedback of what's being deleted recursively
   
   j) rm -d <directory> (or --dir)
      Use Case: Remove empty directories (like rmdir)
      When to use: When you want to remove only empty directories safely
   
   k) find . -name "*.tmp" -type f -exec rm {} \; (with find command)
      Use Case: Remove files found by find command with specific criteria
      When to use: When you want to delete files based on complex conditions (size, age, name pattern, etc.)
   
   l) rm --no-preserve-root /
      Use Case: Override protection against removing root directory
      When to use: System administration only (extremely dangerous!)
   
   m) rm -P <filename> (on some systems)  // ***
      Use Case: Securely remove files by overwriting before deletion
      When to use: When you want to permanently destroy sensitive files so recovery is impossible
   
   n) rm -- <filename>  // ***
      Use Case: Remove files with special or problematic names
      When to use: When filename starts with dash or has special characters
   
   o) rm <file1> <file2> <file3>
      Use Case: Remove multiple specific files at once
      When to use: When you want to delete several known files
   
   p) rm -I <directory> (or --interactive=once)
      Use Case: Prompt once before removing more than 3 files or recursively
      When to use: A middle ground between -i and no prompts for bulk deletions
